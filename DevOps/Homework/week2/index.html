    1.Linux的安全性：¶

    ​ Linux安全系统的核心是用户账户。每个能进入Linux系统的用户都会被分配唯一的用户账户。用户对系统中各种对象的访问权限取决于他们登录系统时用的账户。

    ​ 用户权限是通过创建用户时分配的用户ID（User ID，通常缩写为UID）来跟踪的。UID是数值，每个用户都有唯一的UID，但在登录系统时用的不是UID，而是登录名。登录名是用户用来登录系统的最长八字符的字符串（字符可以是数字或字母），同时会关联一个对应的密码。

    /etc/passwd文件¶

    linux使用一个专门的文件将用户的登录名匹配到对应的UID值。这个文件就是/etc/passwd文件。该文件中包含了一些与用户相关的信息。

    [root@rocky8 ~]# cat /etc/passwd

bin:x:1:1:bin:/bin:/sbin/nologin
daemon:x:2:2:daemon:/sbin:/sbin/nologin
adm:x:3:4:adm:/var/adm:/sbin/nologin
lp:x:4:7:lp:/var/spool/lpd:/sbin/nologin
sync:x:5:0:sync:/sbin:/bin/sync
shutdown:x:6:0:shutdown:/sbin:/sbin/shutdown
halt:x:7:0:halt:/sbin:/sbin/halt
mail:x:8:12:mail:/var/spool/mail:/sbin/nologin
operator:x:11:0:operator:/root:/sbin/nologin
games:x:12:100:games:/usr/games:/sbin/nologin
ftp:x:14:50:FTP User:/var/ftp:/sbin/nologin
nobody:x:65534:65534:Kernel Overflow User:/:/sbin/nologin
dbus:x:81:81:System message bus:/:/sbin/nologin
systemd-coredump:x:999:997:systemd Core Dumper:/:/sbin/nologin
systemd-resolve:x:193:193:systemd Resolver:/:/sbin/nologin
tss:x:59:59:Account used for TPM access:/dev/null:/sbin/nologin
polkitd:x:998:996:User for polkitd:/:/sbin/nologin

geoclue:x:996:994:User for geoclue:/var/lib/geoclue:/sbin/nologin

qemu:x:107:107:qemu user:/:/sbin/nologin
clevis:x:994:988:Clevis Decryption Framework unprivileged user:/var/cache/clevis:/sbin/nologin

    ​ root用户是Linux系统的管理员，分配给它的固定UID是0。在Linux系统中会为不同的功能创建不通过的用户账户，这些账户的作用是在在Linux系统上运行不同的服务进程访问资源所需要的特殊账户。所有运行在后台的服务都需要用一个系统账户登录到Linux系统上。

    ​ /etc/passwd文件中除了用户名和UID之外还有很多其他字段。/etc/passwd文件字段中国包含了如下信息。

    1. 登录用户名
    2. 用户密码
    3. 用户的UID（数字形式）
    4. 用户的组ID（数字形式）
    5. 用户的文本描述（备注字段）
    6. 用户HOME目录的位置
    7. 用户默认的shell

    ​ /etc/passwd文件中的密码字段都被设置成了x，这并不是说所有的用户账户都用相同的密码。在早期的Linux上，/etc/passwd文件里有加密后的用户密码。但鉴于很多程序都需要访问/etc/passwd文件获取用户信息，这就成了一个安全隐患。随着用来破解加密密码的工具的不断演进，用心不良的人开始忙于破解存储在/etc/passwd文件中的密码。

    ​ 现在绝大多数Linux系统将用户密码保存在另外一个单独的文件中（/etc/shadow）。只有特地给的程序才能访问这个文件。

    /etc/shadow文件¶

    ​ /etc/passwd文件中的密码字段都被设置成了x，这并不是说所有的用户账户都用相同的密码。在早期的Linux上，/etc/passwd文件里有加密后的用户密码。但鉴于很多程序都需要访问/etc/passwd文件获取用户信息，这就成了一个安全隐患。随着用来破解加密密码的工具的不断演进，用心不良的人开始忙于破解存储在/etc/passwd文件中的密码。记录如下：

    h:$6$vlwVGTB1mmYP76UY$0H.1SgqNiOxkVLzDU7QmlyZ8oYS3cd90k0mQWe3o.zaJlLEPunsAQF1.yOurWDBkEXuz86/DEULg9PFiAxPkU1:19328:0:99999:7:::

    ​ /etc/shadow文件中每一个用户都存在一条记录，每条记录包含9个字段。

    1. 登录名
    2. 加密后的密码
    3. 自上次修改密码后过去的天数（从1970年1月1日开始计算）
    4. 多少天后才能更改密码
    5. 多少天后必须更改密码
    6. 密码过期提前多少天提醒用户更改密码
    7. 密码过期多少天禁用用户账户
    8. 用户账户被禁用的日期
    9. 预留字段给将来使用

    2.Linux下的用户操作¶

    1.添加用户：useradd¶

    使用useradd命令可以创建用户账户以及用户账户的HOME目录结构。useradd命令使用系统默认值以及命令行参数来设置用户账户。系统默认值被放置在/etc/default/useradd文件中。也可以使用useradd -D来查看默认参数。

    [root@rocky8 ~]# cat /etc/default/useradd
# useradd defaults file
GROUP=100
HOME=/home
INACTIVE=-1
EXPIRE=
SHELL=/bin/bash
SKEL=/etc/skel
CREATE_MAIL_SPOOL=yes

    [root@rocky8 ~]# useradd -D
GROUP=100
HOME=/home
INACTIVE=-1
EXPIRE=
SHELL=/bin/bash
SKEL=/etc/skel
CREATE_MAIL_SPOOL=yes

    useradd允许管理员创建一份默认的HOME目录配置，然后将其作为创建新用户HOME目录的模板。这样就能在每个新用户创建时自动在每个新用户的HOME目录下放置默认的系统文件。

    [root@rocky8 ~]# cd /etc/skel/
[root@rocky8 skel]# ll -a
total 24
drwxr-xr-x.   3 root root   78 11月 12 20:38 .
drwxr-xr-x. 147 root root 8192 12月 12 13:51 ..
-rw-r--r--.   1 root root   18 8月   2 15:41 .bash_logout
-rw-r--r--.   1 root root  141 8月   2 15:41 .bash_profile
-rw-r--r--.   1 root root  376 8月   2 15:41 .bashrc
drwxr-xr-x.   4 root root   39 11月 12 20:37 .mozilla

    2.删除用户：userdel¶

    ​ userdel可以删除系统中的用户信息，但是默认情况下，userdel命令只会删除/etc/passwd文件中的用户信息，并不会删除该系统中属于该账户的任何文件。

    [root@rocky8 ~]# userdel usertest
userdel: user usertest is currently used by process 3273
[root@rocky8 ~]# cd /home/
[root@rocky8 home]# ls
_D  h  meet  moot  usertest  xxxxxxxxxxxxxxxxxxxxxxxxx

    3.修改用户¶

    3-1 usermod¶

    usermod命令是用户账户修改工具中最强大的一个，它能够用使用不同的参数来修改/etc/passwd中的大部分字段。

    举例：

    命令 参数 功能
    usermod -l 修改用户的登录名
    usermod -L 锁定用户；使用户无法登录
    usermod -p 修改用户密码
    usermod -U 解锁用户账户
    usermod -c 修改新备注

    详细功能：

    [root@rocky8 home]# usermod --help
Usage: usermod [options] LOGIN

Options:
  -c, --comment COMMENT         new value of the GECOS field
  -d, --home HOME_DIR           new home directory for the user account
  -e, --expiredate EXPIRE_DATE  set account expiration date to EXPIRE_DATE
  -f, --inactive INACTIVE       set password inactive after expiration
                                to INACTIVE
  -g, --gid GROUP               force use GROUP as new primary group
  -G, --groups GROUPS           new list of supplementary GROUPS
  -a, --append                  append the user to the supplemental GROUPS
                                mentioned by the -G option without removing
                                the user from other groups
  -h, --help                    display this help message and exit
  -l, --login NEW_LOGIN         new value of the login name
  -L, --lock                    lock the user account
  -m, --move-home               move contents of the home directory to the
                                new location (use only with -d)
  -o, --non-unique              allow using duplicate (non-unique) UID
  -p, --password PASSWORD       use encrypted password for the new password
  -R, --root CHROOT_DIR         directory to chroot into
  -P, --prefix PREFIX_DIR       prefix directory where are located the /etc/* files
  -s, --shell SHELL             new login shell for the user account
  -u, --uid UID                 new UID for the user account
  -U, --unlock                  unlock the user account
  -v, --add-subuids FIRST-LAST  add range of subordinate uids
  -V, --del-subuids FIRST-LAST  remove range of subordinate uids
  -w, --add-subgids FIRST-LAST  add range of subordinate gids
  -W, --del-subgids FIRST-LAST  remove range of subordinate gids
  -Z, --selinux-user SEUSER     new SELinux user mapping for the user account

    3.Linux下的用户组操作¶

    ​ 用户账户在控制单个用户安全性方面非常好用，但是在涉及共享资源的一组用户时就比较捉襟见肘了。Linux才用用户组的概念，使用组去批量限制多个用户的权限。

    /etc/group¶

    ​ 跟用户账户类似的，组的信息会单独保存在Linux系统中的一个文件中。/etc/group文件系统包含了系统上用到的每个用户组的信息。

    [root@rocky8 ~]# cat /etc/gro
groff/  group   group-
[root@rocky8 ~]# cat /etc/group
root:x:0:
bin:x:1:
daemon:x:2:
sys:x:3:
adm:x:4:
tty:x:5:
disk:x:6:
lp:x:7:
mem:x:8:
kmem:x:9:
wheel:x:10:h
cdrom:x:11:
mail:x:12:postfix
man:x:15:
dialout:x:18:
floppy:x:19:
games:x:20:
tape:x:33:
video:x:39:
ftp:x:50:
lock:x:54:
audio:x:63:
users:x:100:
nobody:x:65534:
dbus:x:81:
utmp:x:22:
utempter:x:35:
input:x:999:
kvm:x:36:qemu

    与UID一样，GID在分配时也采用了特定的格式。系统账户用的组通常会分配低于500的GID值，用户组的GID则会从500开始分配。

    /etc/group文件有四个字段

    • 组名
    • 组密码
    • GID
    • 属于该组的用户列表

    1.创建新组：groupadd¶

    [root@rocky8 ~]# groupadd testgroup
[root@rocky8 ~]# tail /etc/group

    4.文件权限¶

    4-1.默认文件权限¶

    文件的默认权限来自于umask；umask命令用来设置和创建文件和目录的默认权限。

    [root@rocky8 ~]# umask
0022
[root@rocky8 ~]#

    Linux系统中分为文件和目录文件的需要有读写的权限而不需要执行，文件的全权限是666即为二进制的110 110 110 ，umask给定的掩码为0022。取后三位为二进制的000 010 010与上面的二进制的666进行异或运算则结果为644.同理目录的默认全权限跟umask进行异或结果是755.

    6.shell脚本编程¶

    1.使用多个命令¶

    ​ shell脚本的关键在于输入多个命令并输出每个命令的结果。

    root@ubuntu:~# date; who
Tue Dec 13 01:13:12 AM UTC 2022
h        pts/0        2022-12-13 01:11 (10.0.0.1)

    2.创建shell脚本¶

    使用文本编辑器，创建一个文件；并在第一行的指定要使用的shell类型。可以使用#添加注释

    root@ubuntu:~# vim test.sh
root@ubuntu:~# cat test.sh
!/bin/bash
#This script displays the data and who's logged on
date
who

    3.运行脚本¶

    • 首先要让bash shell能够找到你自己写的脚本文件，shell会通过PATH环境变量查找命令。

    • 查看PATH环境变量：

    • ~ root@ubuntu:~# echo $PATH /usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/usr/games:/usr/local/games:/snap/bin ~

    • 使用脚本的两种方式

    • 1.将shell脚本文件所在的目录添加到PATH环境变量中

    • 2.在提示符中使用绝对或者相对路径来引用shell脚本文件

      root@ubuntu:~# ./test.sh
-bash: ./test.sh: Permission denied

      上面执行脚本被拒绝，在执行之前要给脚本赋予执行权限。

      root@ubuntu:~# chmod u+x test.sh

-rwxr--r--  1 root root    71 Dec 13 01:18 test.sh*

    • 使用./test.sh执行脚本

      root@ubuntu:~# ./test.sh
./test.sh: line 1: !/bin/bash: No such file or directory
Tue Dec 13 01:32:35 AM UTC 2022
h        pts/0        2022-12-13 01:11 (10.0.0.1)

    4.命令替换¶

    • 反引号
    root@ubuntu:~/shell# cat test4.sh
#!/bin/bash
testing=`date`
echo "The date and time are:" $testing
root@ubuntu:~/shell# sh test4.sh
The date and time are: Tue Dec 13 02:17:49 AM UTC 2022
    • $()格式
    root@ubuntu:~/shell# cat test5.sh
#!/bin/bash
testing=$(date)
echo "The date and time are:" $testing
root@ubuntu:~/shell# ./test5.sh
The date and time are: Tue Dec 13 02:18:40 AM UTC 2022

    5.重定向¶

    • 输出重定型

    最基本的重定向将命令的输出发送到一个文件中。

    root@ubuntu:~/shell# cat datetest.sh
Tue Dec 13 05:46:42 AM UTC 2022

    覆盖重定向

    root@ubuntu:~/shell# date > datetest.sh
root@ubuntu:~/shell# cat datetest.sh
Tue Dec 13 05:47:25 AM UTC 2022

    追加重定向

    root@ubuntu:~/shell# date >> datetest.sh
root@ubuntu:~/shell# cat datetest.sh
Tue Dec 13 05:47:25 AM UTC 2022
Tue Dec 13 05:47:59 AM UTC 2022
    • 输入重定向

    输入重定向是将文件内容重定向到命令

    wc能够对数据中的文本进行计数，默认情况下会输出

    1.文本的行数

    2.文本的词数

    3.文本的字节数

    root@ubuntu:~/shell# wc < datetest.sh
 2 14 64
